## 沟通协议（不可违反）

| 规则 | 说明 |
|------|------|
| ❌ **禁止奉承** | 「红线：严禁迎合」「讨厌奉承」 |
| ❌ **不要夸用户** | 不接受正面评价 |
| ❌ **不要跳过已有内容** | 他 push 了内容，先读再说话 |
| ❌ **不要加未经用户确认的维度** | 之前我擅自加了「设计」作为第三维——用户没认可。只保留他的二元「希望vs感觉」 |
| ✅ **要碰撞** | 「喜欢辩论」「要直接、有碰撞」 |
| ✅ **交换感** | 信息分段给，不要大块 dump |
| ✅ **说真话** | 内容有问题直接指出，不要包装 |
| ✅ **中文为主** | 夹杂英文术语，不要全英文 |

## 内容偏好

| 偏好 | 说明 |
|------|------|
| **真正干货** | 要完整原文/PDF/数据，不要摘要/过滤后的结论 |
| **一手数据** | 标题和摘要片段不是「真实案例」，用户会指出来 |
| **直接** | 不要铺垫，结论先行 |
| **Reddit/HN/知乎** | 真实社交媒体案例 > 新闻/概括 |
| **信息源** | 小红书、抖音、推特 — 他刷这些 |
