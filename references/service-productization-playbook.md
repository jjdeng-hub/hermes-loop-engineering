# AI 服务产品化实战手册 — 卖知识，不卖体力

> 来源：2026-06-30 会话中的产品化实践
> 核心原则：客户不需要知道 Hermes，客户只需要拿到结果。

## 核心理念

**不卖工具，卖结果。** 客户不需要安装 Hermes、不需要注册账号、不需要学任何新东西。他们只需要：
1. 告诉我行业 + 关键词
2. 给我一个手机号/邮箱
3. 第二天开始收报告

## 已验证的产品线

### Level 1: 情报系统（¥199 一次性 / ¥99 月付）
- **卖什么**：每天早上 8 点自动推送行业市场报告
- **数据源**：HN Algolia API（可靠、免费、不需要登录）
- **交付方式**：飞书 Bot / 邮件 / PDF
- **客户工作量**：0（VPS 上跑，客户只需收消息）
- **定价逻辑**：¥199 一次性（含 1 周关键词调优）or ¥99/月订阅

### Level 2: AI 员工（¥99-199/月）
- 一个飞书 Bot，绑定一个专长角色（小红书运营、AI程序员等）
- 客户直接在飞书群里跟 Bot 对话
- 背后 = 一个配置好的 Hermes Profile（客户看不见）

### Level 3: AI 团队（¥299-499/月）
- 一个飞书群 + 一个 Bot，能调多个角色干活
- 客户说"帮我写篇笔记"，Bot 内部调趋势研究员 + 内容创作者 + 运营专家
- 客户只看最终结果，看不见内部切换

## 交付架构

```
客户 → 告诉我行业/关键词
  ↓
我在 VPS 上配置（customer_scans/configs/<客户名>.json）
  ↓
每天 cron 自动跑（customer-scan.py <客户名>）
  ↓
报告推送到客户的飞书/邮箱
  ↓
客户只看结果，不管技术
```

## 技术实现

### 扫描脚本
```python
# /root/.hermes/scripts/customer-scan.py <customer_name>
# 读取 /root/customer_scans/configs/<name>.json
# 查询 HN Algolia API
# 输出格式化报告 + 保存到 outputs/
```

### 客户配置格式
```json
{
  "name": "客户名",
  "industry": "行业",
  "keywords": ["关键词1", "关键词2"],
  "delivery": {
    "method": "email",
    "address": "customer@example.com"
  }
}
```

### 添加新客户的流程
1. 创建 `/root/customer_scans/configs/<客户名>.json`
2. 测试运行：`python3 /root/.hermes/scripts/customer-scan.py 客户名`
3. 添加 cron：`30 7 * * * python3 /root/.hermes/scripts/customer-scan.py 客户名`
4. 配置推送渠道（飞书/邮件）

## 定价原则

1. **先验证后扩展** — 先卖 199 的情报系统验证"有人愿意为 AI 服务付费"
2. **订阅制优于一次性** — 月付 99 比 199 一次性更有长期价值
3. **跟客户的深度挂钩** — 关键词越多、行业越专业，价值越大

## 获客渠道

| 渠道 | 优先级 | 怎么做 |
|------|--------|--------|
| 闲鱼 | ✅ 立即 | 挂商品：《AI 每日情报系统——每天早上自动收到行业报告》 |
| 小红书 | 同步进行 | 发"打工人 AI 副业实验"日记，展示真实案例 |
| 朋友圈 | 顺手 | 发一条：帮朋友搭了个情报系统，有人要吗？ |

## 客户沟通要点

- ❌ 不说"Hermes"、"Agent"、"API"
- ✅ 说"每天早上 8 点自动推送到你手机"
- ❌ 不说"你需要注册一个飞书账号..."
- ✅ 问"你平时用飞书还是微信收消息？"

## 关键教训

1. **先卖再验** — 不要等产品完美。闲鱼挂 199 元，有人买就继续，没人买就换方向
2. **非 Hermes 用户是客户主体** — 大多数人不需要（也不应该需要）知道 Hermes
3. **飞书 Bot 是最好的发布渠道** — 免费、稳定、不封号、用户体验好
4. **最小可行产品 = cron + 脚本 + 推送** — 不需要 UI，不需要 App
