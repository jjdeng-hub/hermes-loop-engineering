# 全量配置审查与 Profile 清理

## 背景

之前的 opencode-go provider 路由到 OpenRouter 的问题引发了对整个 Hermes 配置的全面审查。

## 审查结果

### 主模型状态
- Provider: `opencode-go` → 直连 `https://opencode.ai/zen/go/v1`
- 模型: `deepseek-v4-flash`
- API key: 实测 200 OK
- Fallback 链: `sn-sensenova/glm-5.2 → sn-sensenova/deepseek-v4-flash → apikey-fun/gpt-5.4`

### 之前路由到 OpenRouter 的原因
配置中 provider 未显式绑定到 `opencode-go` 时，`deepseek-v4-flash` 这个模型名会被 Hermes 模型目录（model_catalog）匹配到 OpenRouter 的 `deepseek/deepseek-v4-flash` 入口。切换 provider 到 `opencode-go` 即修复。

| opencode.ai | ✅ 200 |
| token.sensenova.cn | ✅ 200 |

| api.deepseek.com | ✅ 可达 |
| openrouter.ai | ✅ 可达 |

### Cron Jobs (4个)
全部 pinned 到 `opencode-go/deepseek-v4-flash`，近期均成功运行：
- daily-backup (3am → feishu)
- hermes-auto-update (6am → feishu)
- vault-backup (每6h → local)
- daily-opportunity-scan (7am → feishu)
