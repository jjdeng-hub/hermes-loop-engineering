# 每日报告改造

## 背景

原来的 `daily-opportunity-scan` cron 每天早上 7 点发报告，包含：
- 机器状态（永远正常，没用）
- HN 商机扫描（反爬拦截、内容滞后）
- Token 用量（Jerry 已放弃）
- 推荐行动（从未执行）

Jerry 说"这些内容我们现在也用不到"，要求改成 AI 资讯方向。

## 改了什么

### 新脚本：`/root/.hermes/scripts/github-trending-scan.py`

替代旧的 `opportunity-scan.py`（HN Algolia 查询），改为采集：
- **GitHub 日榜**（trending?since=daily）— 爬页面，提取 repo 名、描述、今日 star、语言
- **GitHub 周榜**（trending?since=weekly）— 同上，补漏
- **GitHub 搜索**（search/repositories API）— 按 `created:>=today AI/agent/llm/gpt` 查询，按 star 排序

输出结构化 JSON，LLM 后续做过滤和分析。

### 新报告结构

```
# ☀️ AI 简报 · 2026-07-04

## 📦 GitHub 日榜 · AI 相关
Top 5-8 项目，每个标注：
- 简介
- 用得上？→ ✅ / 🔧 / 👀
- 能干嘛

## 🔥 本周值得关注的新项目
（搜索 API 来的，不在日榜上但质量高）

## ✅ 今日可做
1-3 个具体行动
```

### Cron 更新
- 任务名：`daily-opportunity-scan` → `daily-ai-trending`
- 调度：不变（`0 7 * * *`）
- 投递：不变（飞书 Jerry 主页）
- 模型：不变（opencode-go/deepseek-v4-flash）

## 核心逻辑

- 过滤掉 elasticsearch、maven、ansible、pytorch 等老牌项目（除非重大更新）
- 只关注 AI / 开发工具 / 内容创作相关
- 每个项目标「能用吗」
- 今天没值得说的就报「无特别发现」，不硬凑
