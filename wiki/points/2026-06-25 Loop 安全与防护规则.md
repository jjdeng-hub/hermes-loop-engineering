# Loop 安全与防护规则

## 路径黑名单

Loop **绝不能**未经人类批准自动编辑这些路径：
```
.env, .env.*
**/secrets/**、**/credentials/**
**/*_key*、**/*_secret*
.terraform/**、k8s/production/**
auth/**、payments/**、billing/**
```

## 自动合并策略

**默认：不自动合并。**

如果允许自动合并，仅限：
- 注释/文档中的错别字 ✅
- 测试文件中的 lint 自动修复 ✅
- 导入顺序调整 ✅
- allowlisted `docs/` 路径中的配置 ✅

不允许：
- 行为变更 ❌
- 依赖版本更新 ❌
- 锁文件变化 ❌
- 任何黑名单路径 ❌

## MCP 连接器最小权限

| 连接器 | 可读 | 可写 |
|--------|------|------|
| GitHub | issues, PRs, checks | comment, label（默认不能 merge） |
| Linear | 团队 issues | comment, status（不能删除） |
| Slack | 频道历史 | 只能发到 `#loop-escalations` |
| 数据库 | — | 不能从 loop 写入生产库 |

## 必须人工门控的操作

- 安全、认证、授权
- 支付、计费、PII 处理
- 基础设施 / Terraform / K8s 生产环境
- 依赖升级（供应链风险）
- 变更涉及超过 N 个文件（建议 N=10）
- 同一项目的第三次失败尝试

## 秘密信息保护

- 绝不在调度提示词中粘贴 API Key
- CI 日志可能包含秘密 — triage skill 应在写入状态前脱敏
- 状态文件通常被提交 — `STATE.md` 中不能有凭证

## 事故响应

如果 loop 合并了坏代码：
1. 立即暂停所有 loop（`scheduler_list` → delete）
2. 回滚合并
3. 记录到状态 + `stories/`
4. 收紧验证器或缩小范围后重启
