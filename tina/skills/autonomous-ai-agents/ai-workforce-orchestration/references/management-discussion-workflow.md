# 管理层讨论流程

> 创建于 2026-06-25，基于 Jerry 产出的完整讨论流程。
> 此文档是 `ai-workforce-orchestration` 的参考文件。
> 管理层的 6 个角色：Demon（战略）、Alex（PM）、Allen（高级PM）、Jack（牧羊人）、Karen（制片人）、趋势研究员（市场分析）。

## 触发条件

Tom1 收到 Jerry 的议题后启动。议题可以是：

- "我要做知识库产品，管理层讨论一下方向和策略"
- "我们收到一个客户需求，评估要不要接"
- "Q3 战略方向需要确定，请管理层出建议"

## 异步设计原则

- **不是所有人围一桌开会**。这些是 AI agent，不是真人。
- **每个人都产出自已领域内的判断**，不是讨论稿。
- **分阶段执行**：上一阶段的输出是下一阶段的输入。

## 阶段一：信息输入

Tom1 先做两件事：

1. 把议题 + 背景打包
2. 判断是否需要市场数据 → 如果需要，发给**趋势研究员**做研究
3. 趋势研究员出报告 → 回到 Tom1

如果没有市场研究需求，直接从阶段二开始。

## 阶段二：分角色输入（并行）

Tom1 把议题 + 趋势研究员的报告，同时发给所有管理层成员。每人产出自己领域的判断。

### 角色分工

| 角色 | 核心问题 | 输出物 |
|------|---------|--------|
| **Karen**（制片人） | 这事跟公司战略一致吗？ROI 大概多少？值得投入吗？要不要做？ | 战略评估 + ROI 估算 |
| **Alex**（PM） | 用户真正需要什么？方向怎么定？优先级怎么排？ | 方向建议 + 优先级排序 |
| **Jack**（牧羊人） | 要做多大？关键路径是什么？什么风险？多长时间？ | 范围框定 + 时间线初版 + 风险清单 |
| **Demon**（战略搭档） | 前提假设是什么？我们漏了什么？有没有更好的路径？ | 挑战清单 + 替代方案 |
| **Allen**（高级PM） | 如果做，任务怎么拆？资源怎么配？ | 任务分层草案 + 资源评估 |

### 关键规则

- **不是投票** — 分歧点 Tom1 做标注，Jerry 做最终判断
- **不重复劳动** — 上一阶段的输出下游可以参考，不需要等全部到齐
- **每人只产出自己领域的判断** — Alex 不写 ROI，Karen 不拆任务

## 阶段三：冲突识别与对齐

Tom1 收集所有输入，做三件事：

1. **标记共识区域** → 大家都同意的地方
2. **标记分歧区域** → 有人说不有人说行
3. **标记空白区域** → 所有人都没提到但很重要的问题

### 第二轮聚焦讨论

仅在存在重大分歧时触发。Tom1 只针对分歧点发给对应角色回应：

> 举例：Karen 说 ROI 低不值得做，Alex 说市场有需求值得做
> → Tom1 发回去让两人各自补充数据支撑

## 阶段四：合成建议

Tom1 把对齐后的结果合成为一份**管理层建议**，使用固定格式：

```
议题：[一句话]

共识结论：
- ...
- ...

待定事项（Jerry 需要决策）：
- ...

建议方案：
- 推荐路径 + 理由
- 替代路径 + 理由

建议下一步：
- 如果 Jerry 批准 → Allen 拆任务，分配给执行层
- 如果 Jerry 有疑问 → 管理层补充研究
```

## 阶段五：Jerry 决策

Tom1 把管理层建议发给 Jerry。Jerry 三种反应：

- **批准** → Tom1 通知 Allen 拆任务，启动执行流水线
- **驳回/调整方向** → Tom1 把新方向返回阶段二，只影响相关角色
- **有疑问** → Tom1 把疑问转给对应角色补充

## 存档要求

每次讨论的输出物存档到 Obsidian `4-Tom-Memory/Decisions/`，包含：
- 议题
- 各角色的判断摘要
- 分歧点
- Jerry 的决策 + 理由

便于后续复盘"当初为什么这么定"。
