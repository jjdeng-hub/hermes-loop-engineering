## 错误码含义

| 错误码 | 含义 | 应对 |
|--------|------|------|
| 429 RateLimit | 限流——短时间内请求太多 | 等几分钟再试，或切 provider |
| 503 Inference temporarily unavailable | 服务不可用——后端负载过高或宕机 | 切 provider，等 30min+ 再试 |
| failover_exhausted | provider 内部所有兜底链路都挂了 | 短期内不会恢复，直接切 provider |

## 根因分析

opencode-go 是一个 **聚合代理**（aggregation proxy），它内部路由到各个模型厂商的 API。当它返回 503 时，意味着其自身或上游链路挂了。前 3 个 fallback 也使用同一个聚合代理，所以一起挂掉。

## 教训

1. **永远不要把全部 fallback 放在同一个 provider 上。** 聚合代理挂掉时，所有路由到它的模型同时失效。
2. **Provider 切换后需要 `/new` 重启会话。** 当前 session 带着旧 provider 的上下文继续重试，不会自动切 fallback。
3. **商汤 sensenova 是可靠的国内备选。** 2026-07-02 整晚测试，deepseek-v4-flash 和 glm-5.2 一直稳定返回 HTTP 200。
