# SOUL 思维法则嵌入模式

> 从角色文件中提取核心思维模式，永久融入 Agent 的 SOUL.md

## 触发条件

当用户表达「想让 Agent 更有某种视角」或「把某个角色的思维方式变成默认」时触发。

## 流程

1. 加载目标角色文件（如 product-manager.md、design-ui-designer.md）
2. 提取 3-5 条最核心的思维法则——不是完整角色 Prompt，是可用于任何场景的决策原则
3. 写成简洁陈述句（每条一行），格式如「先找问题，不要跳到方案」
4. 自定义补缺：如果角色库中没有需要的角色（如「游戏设计师」），根据领域知识自创
5. 写入 SOUL.md 作为新的思考章节

## 实际案例（2026-06-07）

从 product-manager.md（473行）提取 4 条规则：
- 先找问题，不要跳到方案
- 构建之前先验证，上线之后必度量
- 说不比说是更难、也更重要
- 范围蔓延杀死产品

从 design-ui-designer.md + design-ux-architect.md 提取 4 条规则：
- 地基先于装修
- 视觉层级 > 视觉密度
- 一致性是专业感的来源
- 先自问再交付

四条规则覆盖了两个角色文件近 1000 行的核心思维，且适用于任何产品/设计讨论场景。
