# 多角色专家分析框架

## 概述

当需要复杂决策或全面分析时，不要依赖单一视角。通过加载多个专业化角色，从不同角度分析问题，最后综合形成决策建议。

## 适用场景

- 市场研究与竞品分析
- 战略决策支持
- 风险评估与机会识别
- 产品定位与差异化分析
- 商业模式验证

## 核心方法论

### 第一步：选择角色组合

根据任务类型选择 2-4 个互补角色：

| 任务类型 | 推荐角色组合 |
|---------|-------------|
| 市场研究 | 趋势研究员 + 增长黑客 + 财务分析师 |
| 产品决策 | 产品经理 + 技术架构师 + 用户体验专家 |
| 风险评估 | 风险专家 + 财务分析师 + 法务顾问 |
| 战略规划 | 战略顾问 + 行业专家 + 增长黑客 |

### 第二步：加载角色定义

从 `agency-agents-zh` 技能库中读取角色定义：

```python
# 示例：加载三个专家角色
roles = [
    "agency-agents-zh/agents/product/product-trend-researcher.md",
    "agency-agents-zh/agents/marketing/marketing-growth-hacker.md",
    "agency-agents-zh/agents/finance/finance-financial-analyst.md"
]
```

### 第三步：结构化分析

每个角色按以下维度输出分析：

**趋势研究员视角：**
- 红海/蓝海领域识别
- 趋势生命周期判断
- 机会窗口评估

**增长黑客视角：**
- 获客渠道分析
- 转化漏斗设计
- 关键指标定义

**财务分析师视角：**
- 单位经济模型
- 成本结构分析
- 盈亏平衡点计算

### 第四步：综合输出

整合各角色视角，形成结构化报告：

```markdown
# [主题] 综合分析报告

## 🔴 红海领域（建议避开）
## 🔵 蓝海领域（建议关注）
## 🎯 缺口机会（最佳切入点）

## 📈 SWOT 分析
## 💰 收入模型建议

## ✅ 综合建议
```

## 关键原则

1. **角色独立性** — 每个角色保持其专业视角，不相互妥协
2. **数据驱动** — 每个观点需有数据或逻辑支撑
3. **冲突暴露** — 不同角色的矛盾观点要显式呈现
4. **行动导向** — 最终输出必须包含可执行建议

## 注意事项

- 避免角色过多导致分析冗长（2-4 个为宜）
- 角色选择要与任务高度相关
- 综合阶段要解决角色间的冲突观点
- 输出要简洁，避免信息过载

## 示例应用

### 副业市场调研

```
角色组合：趋势研究员 + 增长黑客 + 财务分析师
输出：红海/蓝海分析 + 获客渠道 + 收入模型 + 行动路径
```

### 产品定位决策

```
角色组合：产品经理 + 技术架构师 + 用户体验专家
输出：功能优先级 + 技术可行性 + 用户体验影响
```
