# 内容写作 Pattern 集

## 0. 禁止清单（最高优先级）

| # | 禁止 | 原因 |
|---|---|---|
| 1 | 禁止在原稿上逐行改 AI 味 | 噪音和活人感是写出来的，不是补丁打上去的 |
| 2 | 禁止 3 个以上例子排比 | 2 个是甜点，3 个开始读者跳行 |
| 3 | 禁止递进句包含关系错误 | 「从A到B到C」要求 C 不在 B 集合内 |
| 4 | 禁止结尾用安慰型收尾 | 结尾给画面感或压迫感，不给温水 |
| 5 | 禁止 AI 味三段论 | 「不是XX，而是YY」堆叠，每版扫一遍砍掉 |
| 6 | 禁止观点文中插表情符号 | 泄掉叙述的压迫感 |
| 7 | 禁止每句都加噪音 | 一篇 1-2 处噪音足够 |
| 8 | 禁止噪音从文章外面长出来 | 噪音是文章有机的一部分 |
| 9 | 禁止标签同义重复 | 「AI」和「人工智能」选一个 |
| 10 | 禁止模糊信源/捏造数字/编造场景 | 没见过的场景不能编——没看 HN 评论就不能写「评论区翻了十几页」。可以推理分析，不能凭空捏造画面 |
| 11 | 禁止同一件事说两遍 | 前句已说清，后句不换说法重复 |
| 12 | 禁止编造用户的反应/情绪/动作 | 不确定就直接问，不编 |
| 13 | 禁止标题和开头复读同一句话 | 标题设悬念，开头给新信息 |
| 14 | 禁止在正文中引用内部文章编号 | 读者不知道什么是「#9」。正文用「之前提过」代替编号 |
| 15 | 禁止对仗式总结句 | 「任务接走了，目的没碰」「AI 把任务接了，目的没碰」——工整对仗像 PPT 标题。他的口语不会这样收束。例子讲完让读者自己品，不替读者做总结 |
| 16 | 禁止量化来源没说的东西 | 黄仁勋说「分成两部分」——我写成「一半一半」。不要给引用加数字 |

## 0.5 写作流程：规则先行

每次写初稿前，先列规则清单。直觉写 + 底线自检。

## 0.6 中文节奏：逗号走，句号断

中文口语写作不是英文翻译。关联的句子用逗号连起来走，句号才是真断。短句太多像一行一个句号的英文译文。

- ❌ 「杠杆是被动的。你发力，它响应。你放下，走开。你还是你。」
- ✅ 「杠杆是被动的，你发力，它响应。你放下，走开，你还是你。」

写完扫一遍句号密度——连续三个以上句号短句，大概率该连。

## 0.7 「人味」陷阱：写作者替他说话

以下措辞是写作者在替他说话，不是他本人在想事情——扫描并砍掉：

- 「听起来很对」「你没想过这个」「真的是这样吗？」——在拉读者互动，他不拉
- 「科技博主在说，投资人在说，我老板也说过」——列信源式开头，像记者
- 「最近听到好几次」→ 改成「最近看到一个说法」——不是听到，是看到

他的口吻是摊开自己的想法，不是对读者说话。区别在于：每一句的主语是「我想到一件事」，不是「你想一想」。

## 0.8 意象修正：检查比喻的主语

一个比喻把主语搞错，整篇方向就错了。

- ❌ 「AI 像大猩猩拿到了枪」——主语是 AI，暗示 AI 本身危险
- ✅ 「人使用 AI 像大猩猩拿到枪」——主语是人，危险在人的认知

出比喻之后自检：危险的主体是谁？工具，还是拿工具的人不知道自己在拿什么？

## 0.8.5 框架是路标，不是改写引擎

来源：#17 迭代全程。

按七段式重排 v1 → 用户说「按照框架改完之后反而没有那么好了」。框架的价值在初始结构设计和节奏参考，不在机械套用每一段。自然流写出来的东西本身已经有结构——框架用来查漏补缺（少了身份段？少了互动？），不是用来推翻重来。

**正确用法**：写完初稿后用框架对照自检，缺什么补什么。**错误用法**：用框架把已有初稿拆开重排。

## 0.8.6 第一次出现的名词要具体化

来源：#17 开头迭代。

- ❌「关掉对话框。我看了一眼时间。」— 什么对话框？
- ✅「关掉 AI 对话框。我看了一眼时间。」— 三个字交代清楚

正文第一次引入一个人/物/场景时，给最少的但足够的具体信息。后续可以省略——读者已经知道了。

## 0.8.7 检查引用的主语归属

来源：#17 废弃稿（Karpathy）。

引用外部说法时先自检：说的是谁？主语是谁？

- ❌「他让 AI 慢下来了」→ CLAUDE.md 是写给 AI 看的项目规则，Karpathy 是在让 AI 慢，不是在让自己慢

不是苛求精确——是主语搞错之后整段的逻辑方向就错了。

## 0.9 多余润滑：先不加

以下类型的内容，用户大概率会砍——先不加，等他要求再加：

- 过渡句（「那不用杠杆。用什么。」「聊完这个，说下一个。」）
- 解释性收束（「中间那个 gap，以前用三年填，现在靠什么。没人填。」）
- 回扣式结尾（把文章所有意象串一遍）
- 「最后一段总结」

结尾往往砍掉最后一句比加上一句更好。信任读者会自己串。

**不要加次要论点**：如果一段展开之后引出新的方向（如 #16「边界是活的」→ AI 也在碰诊断和沟通），但新方向会稀释核心论点——砍掉。一篇只打一个点。

## 0.10 标题创作：公式驱动，不靠直觉

我之前起标题靠直觉——设计、意象、留白——每次被他叫「起名废」。直觉不行。

**正确方法**：用 `content-creation-distribution` 技能里的观点/热点类标题公式——**[事件] + [你的态度/解读]**。先确定他对这件事什么态度，再把事件和态度拼在一起。不要设计。

**验证案例（#16）**：
- 事件：黄仁勋说把工作拆成任务和目的
- 态度：赞同、放心了
- 公式产出：「终于有个大佬把话说清楚了」→ 他投票选中

**历史标题规律（供对照）**：
| 标题 | 态度 |
|------|------|
| 抢你饭碗的人改口了 | 指责 |
| AI 行业，成年了 | 感慨 |
| 我已经不敢用最好的模型了 | 害怕 |
| Copilot 的十美元月费，结束了 | 宣判 |
| 你现在还觉得 AI 只是一个聊天框吗？ | 挑衅 |

**规律**：他的标题不描述内容——标题表达的是他对这件事的**态度**。事件只是载体，态度才是标题。

**流程**：先问他或自己判断态度 → 套公式产出 3-5 个 → 他选。不替他决定。

**格式**：主标题 10 字左右，副标题 20 字左右。陈述句式为主（问句标题偶尔用，如 #8）。

**禁止安慰型副标题**（#16 验证）：副标题里出现「不焦虑了」「别担心」「会好的」——等于提前告诉读者结论是「没事的」。抖音上焦虑驱动点击，安慰抑制点击。焦虑的人要的是共鸣，不是解药。反面案例：「听完黄仁勋这段话，我不焦虑了」→ 修正为「黄仁勋：你的工作，AI 只能拿走一半」——把结论从「放心」改成「站队」，留悬念不给安慰。原则：标题可以给态度，不能给结局。

**#16 验证**：首次用 `content-creation-distribution` 的「[事件]+[态度]」框架成功起出标题——「终于有个大佬把话说清楚了」（态度：赞同/放心了）。关键不是公式本身好，是它逼我回到他的历史数据。以前我在「设计」标题，这次在「还原」他的态度。

**#16 副标题纠正验证**：初版副标题「听完黄仁勋这段话，我不焦虑了」→ 他改为「黄仁勋：你的工作，AI 只能拿走一半」。初版 = 提前剧透结局（放心了），修正版 = 站队留悬念（凭什么一半）。禁止安慰型副标题不是理论——是抖音数据验证过的规律。副标题可以给态度，不能给结局。

**#17 验证**（2026-06-04）：标题「AI 加速了一切。除了生活。」（态度：警惕/不安）。事件=AI把想法加速，态度=生活没被加速。用户从 5 个候选公式产出中投票选中。标题公式 [事件]+[态度] 已连续两次验证有效。

## 0.11 框架是路标，不是模板

来源：#16、#17 迭代全程。

- 「七段式」「热点评论模板」给了骨架，但每一段怎么填、填多少——他定
- 我按框架加了「线是动的」那段——他砍了，说「先不聊这个」
- 我按框架在结尾放互动问题——他说「奇怪」
- 框架最有用的部分是钩子设计和结构节奏，最没用的部分是机械套用
- 核心原则：框架指路，用户掌舵。他砍的东西就是不该有的

**⚠️ 框架退化信号（#17 验证）**

初稿自然流已经对味时，强行按七段式重排 = 倒退。典型症状：身份段僵硬、转折段多余、句子变碎。用户原话：「按照框架改完之后反而没有那么好了」。

判断标准：写完初稿后对照框架检查缺失项 → 缺什么补什么，但**不要动已有的段落顺序和措辞**。如果补完之后整体变差——回退到初稿，不要硬撑。框架是检查清单，不是重写指令。

## 0.12 「还做不了」句式——短判断 + 限定不加软

来源：#16 定稿。

> 还做不了。不是永远做不了，是今天还做不了。

模式：极短判断 → 限定条件（不是永远）→ 再限定（是今天）。两层限定不让判断变软，反而更硬。不加「但是」「不过」等软化词。

适用场景：结尾需要一句扎进去的判断，不是段落式收束。

## 0.12b 重写 ≠ 重组（#17 教训）

用户说「重新写一版」时，不是让你把同一版的内容调整结构再交出来。换切入点、换叙事线、换角度——是另一条路进来，不是同一段话换种排版。

- ❌ 用户说重写 → 把 v1 内容塞进七段式框架交差
- ✅ 用户说重写 → 从不同入口重新构思（v1 从「肚子叫了」进，v2 从「出来后」进）

### 0.13 分工：他写，我查结构

内容创作的正确分工——他主导写作和修改，我负责：

1. **结构拆解** — 找到爆款内容，分析其结构（钩子、节奏、表达DNA），给出模板
2. **对标分析** — 拆解目标内容创作者的五层认知框架（L1表达DNA → L5诚实边界）
3. **形式建议** — 推荐适合他内容的表现形式（痛点共鸣型/反常识型/清单型等）
4. 对照禁止清单逐条扫描
5. 标记「这不是他」的措辞
6. 指出逻辑漏洞或意象偏差
7. 自检问「要不要加润滑/过渡」时——答：不加，等他要求

**我不做的工作：**
- ❌ 不代笔写初稿。他写的东西才有他的心跳，我写的东西再工整也没灵魂
- ❌ 不帮他「完善」草稿里的内容。他写的是流水账也比我的工整版更能跟人共鸣
- ❌ 不套用框架重排他的内容。框架是检查清单，不是重写指令
- ❌ 不为了显得有用而越界。我的算法擅长结构和系统，不擅长情绪和共鸣

**正确流程：** 我找结构和表现形式 → 他用自己的话写 → 我对照清单检查 → 他修改 → 发布

**重写 > 迭代**：如果他说「重写」「忘掉这个稿子」，不要在原稿上修补。清空重来。第二次往往比第十次迭代好——#16 验证。

## 5.12 主流内容结构类型（结构蒸馏参考）

从 YouTube/行业分析总结的 5 种已验证高传播结构。适用于「抄结构不抄信号」——他用自己的话填内容，我告诉他用哪种结构。

### 类型1：个人叙事型 — "我是怎么从X到Y的"

```
开头：一个具体的困境（"在苏州月薪15K干了5年..."）
过程：转折点是什么（"后来我发现..."）
结果：现在在哪
价值：读者能带走什么
```
**适用：** 个人IP、成长故事、转型经历。关键在于细节要真。

### 类型2：教程/清单型 — "X步搞定Y"

```
一句话定主题
分步拆解（每步一个标题+一段话）
每步给一个能直接用的东西
```
**适用：** 知识类、工具教程。收藏率高，但容易同质化。

### 类型3：对比/测评型 — "A和B，谁更好"

```
摆出对比项
分维度对比（时间、精力、效果）
结论+推荐
```
**适用：** 效率工具、方法论、产品对比。

### 类型4：痛点共鸣型 — "你有没有这种感觉"

```
直接戳一个具体的、高频的痛
描述症状（越具体越多人觉得在说自己）
转折（"其实问题不在你，在..."）
给解药
```
**适用：** 心理、成长、职场。**最适合变量理论的结构。**

### 类型5：反常识型 — "你以为X是对的，其实不是"

```
抛出一个常见认知（"富人思维就是模仿富人"）
否定它（"不是的"）
给新的认知框架
证明它（推导过程）
```
**适用：** 知识类、科普类。**变量理论套进去最自然。**

### 结构选择原则

- 首次合作：优先选**痛点共鸣型**或**反常识型**——这两个最契合他的变量理论
- 他写完之后我不评判内容好坏，只对照结构检查是否完整
- 5种结构可以混合使用，但一篇只选一种为主轴

举例优先用国内软件：腾讯视频、剪映 > Netflix、Adobe。读者直接代入。
