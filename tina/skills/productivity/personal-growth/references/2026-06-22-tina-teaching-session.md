## 新增教学领域

### 1. 移动网络/流量/5G

**触发点**：Jerry 问"流量是怎么产生的"

**锚点**：从 Jerry 已知的公司内网 TCP/IP 入手
- 你工位有线网 = 网口 + 交换机 + NAT 网关
- 手机移动网络 = 基站（天线）+ 核心网 + 公网互联网
- 核心概念链：无线电波 → 空口加密 → 频段/时隙/资源块 → 流量计费本质是买"占用空中通道的时间"

**关键技术点**：
- 基站只是天线，后面连核心网（认证+计费+IP分配）
- 手机 IP = 运营商内网 CGNAT 地址（100.x.x.x），多人共用一个公网 IP
- 流量本质 = 你占用无线资源的时间 × 传输速率（不是"数据量"）
- 4G vs 5G = 三个维度升级（带宽 10x + 时延 50x + 并发 500x），不止提速

**Jerry 的顿悟**："我一直以为流量就是我的手机能够在网络上处理的数据量" → 明白了流量本质是"时间"而非"数据"

### 2. 流量卡灰色产业链

**触发点**：Jerry 追问"9块9 100G的卡怎么做到的"

**三类流量卡操作方式**：
1. **物联网卡**（最常见）— 批发价 0.1-0.3 元/G，严禁在手机上用，后台能检测 IMEI
2. **校园/政企卡** — 优惠期 6-12 个月，到期自动变原价
3. **日租/定向卡** — 小字陷阱，通用流量不够用

**核心洞察**：运营商视角——边际成本≈0，灰色产业链帮出货，出事后甩锅

### 3. 《失控》九律框架

**触发点**：Jerry 看到推荐想理解"涌现"和整本书

**教学方法**：
- 不以"我告诉你九律是什么"开讲——先让 Jerry 自己说出涌现的例子（一行行代码堆出复杂系统行为）
- 用 Bing 搜索验证九律原文（网络受限，未完全成功——坦白告诉了 Jerry）
- 用自己的记忆提供九条法则，标注不确定性
- 让 Jerry 自己评估每条对一人公司的适用性（他排除了第8条"持续不均衡"）

**关键对话模式**：Jerry 说"除了8，其他都适用" → 追问"为什么排除8？" → Jerry 自己说出"我希望公司是有序的"

### 4. 公司架构模块化生长

**触发点**：从九律第4条"模块化生长"切入

**Jerry 自己走的诊断路径**：
- "角色分工不够清晰" → "哪个角色边界最模糊？" → "Tom1"
- "Tom1 在做什么不该做的事？" → "替代程序员做开发，替代项目经理做分工"
- "这些事应该谁做？" → "Justin 和 eli"
- "那为什么 Tom1 还在做？" → "习惯了通过 Tom1 一条线出去"

**根因诊断**：不是分工问题，是**路径依赖问题**。
