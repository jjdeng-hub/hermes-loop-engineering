# 知行合一破解工作流

## 来源

2026-06-23 Tina-Session，Jerry 自己生成的认知框架。

## 核心洞察

"知道"和"做到"之间的鸿沟，不是因为懒，而是因为"知道"是一个集合——

> **你知道其中几个子集，但不知道全部，所以动不了。**

## 刷牙类比

- "想刷牙" = 知道大道理（知的上层）
- "不知道怎么挤牙膏、接水、漱口" = 缺少子步骤（知的下面几层）
- 结果：刷不了牙

## 破解步骤

当 Jerry 说"我想做 X 但行动不了"时：

```
第1步：把 X 拆成"已经知道怎么做的子步骤"和"不知道怎么做的子步骤"
第2步：对"不知道怎么做的子步骤"问：有没有办法通过搜索/观察/试错来补上？
第3步：对"已经知道的子步骤"——立即确认其中第一步是什么
第4步：只做这第一步
```

## 案例：卖知识库

Jerry 的拆解过程：

| 已经知道 | 不知道 | 补救方法 |
|---------|--------|---------|
| 要整理成一份文件，分类，目录 | 有没有市场？ | 去闲鱼搜同类商品 |
| 要展示图 | 怎么定价？ | 有同行参考定价，没有就上架测 |
| 要介绍文案 | 文件怎么交付？ | 先做前面几步再考虑 |

## 适用条件

- Jerry 因为"不知道怎么做"而卡住，不是因为"不想做"
- 目标可以被拆成具体的子步骤
- 不需要在第一步之前就解决所有"不知道"
