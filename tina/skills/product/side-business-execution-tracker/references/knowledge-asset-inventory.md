# 知识资产盘点方法论

## 适用场景
- 用户有知识积累（文章、文档、笔记、经验）但不知道能卖什么
- 需要从"先建产品再找客户"切换为"先盘点资产再决定卖什么"
- 知识型副业的启动阶段：在动手做产品之前，先搞清楚自己有什么

## 核心原则

**先盘点，再决定。不是先建产品再找客户，也不是先找客户再建产品——先搞清楚自己手里有什么牌。**

一个人的知识资产 = 公开可收集的内容 + 个人独有经验。盘点的目标是画出知识地图，找到：
1. 哪些领域已覆盖（可做内容/产品）
2. 哪些领域是空白（是机会还是陷阱？）
3. 哪些是个人独有经验（不可替代的护城河）

## 三步盘点法

### 第一步：收集全部知识素材
- 已爬取的文章/文档（如公众号文章、技术博客、行业报告）
- 个人笔记/经验总结
- 行业书籍/标准文档（SECS/GEM 标准、设备手册等）
- 客户问题/常见故障记录

将所有素材统一格式（Markdown 最佳），放在同一目录下。

### 第二步：关键词覆盖分析

定义知识领域的关键词组，扫描所有文章统计提及次数。代码模板：

```python
import os, re
from collections import Counter

keyword_groups = {
    '领域A': [r'关键词1|keyword1', r'关键词2|keyword2'],
    '领域B': [r'关键词3|keyword3'],
    # ... 根据你的行业定义
}

all_terms = Counter()
for fpath in md_files:
    with open(fpath, 'r', encoding='utf-8') as f:
        content = f.read()
    for group, patterns in keyword_groups.items():
        for pat in patterns:
            matches = re.findall(pat, content, re.IGNORECASE)
            if matches:
                all_terms[group] += len(matches)

# 打印覆盖柱状图
for term, count in all_terms.most_common():
    bar = '█' * (count // 20)
    print(f"  {term:20s} {bar} {count}")
```

关键词组要基于你对行业的理解来定义——不用太细，10-15 个领域即可。

### 第三步：绘制知识地图

盘点结果输出为一张知识地图，回答三个问题：

| 问题 | 分析维度 |
|------|---------|
| **覆盖了什么？** | 高覆盖领域 → 可以做内容/培训/工具 |
| **缺了什么？** | 低覆盖领域 → 是市场不需要，还是公开信息稀缺？ |
| **护城河在哪？** | 个人独有经验 vs 公开可获取 → 前者是真正的壁垒 |

**关键洞察：低覆盖 ≠ 没价值。如果某个领域（如 SECS/GEM）在公开内容中几乎没有提及，但它恰恰是你的专业——这就是你的护城河。**

## 从知识地图到产品决策

盘点完成后，不是立刻做产品，而是：

1. **确认知识完整性**：还有没有其他来源的文章/资料可以补充？
2. **合并多来源**：等所有素材到齐后再做一次完整盘点
3. **对照市场需求**：用盘点结果匹配之前调研的市场需求信号
4. **选第一个产品**：挑一个「覆盖好 + 市场有需求 + 你能快速交付」的切口

## 实战案例：半导体焊线机知识库盘点

- **素材**：142 篇公众号文章，16 万字
- **覆盖领域**：14 个（焊线工艺、软件操作、参数配置、校准、报警排查等）
- **最大缺口**：SECS/GEM 通信协议（仅 1 次提及）
- **缺口解读**：这不是坏消息——说明公开内容没人写 SECS/GEM，而用户恰恰是 SECS/GEM 专家。缺口 = 独占机会。

## 与 skill 其他子系统的关系

- **市场调研子系统**（job-market-research.md）：盘点告诉你有什么，市场调研告诉你需要什么。两者交集 → 产品方向。
- **服务菜单生成子系统**：盘点结果直接输入服务菜单的六维评估框架。
- **方向危机处理**（direction-elimination.md）：当用户质疑方向时，回到知识地图——可能不是方向错了，是还没把缺的知识补上。
