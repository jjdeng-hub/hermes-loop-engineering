# 服务菜单生成方法论

> 从专家角色库 + 技能库 → 可售卖服务菜单的完整流程

## 完整流程

### 第一步：技能盘点

列出当前可用的所有能力来源：

| 来源 | 数量 | 示例 |
|------|------|------|
| 角色库（agency-agents-zh） | 214 个专家角色 | AI工程师、小红书运营、图像提示工程师 |
| Hermes Skills | 130 个技能 | baoyu-infographic、enterprise-rag-kb、powerpoint |
| 用户自身能力 | 用户特定 | C++/Python、封测行业知识 |

### 第二步：从每个角色提取「可交付物」

打开每个角色的 .md 文件（`agents/<department>/<role>.md`），在 frontmatter 和 body 中找：

- 角色描述中有没有提到「输出物」「交付物」
- 角色关键词（如「趋势追踪」→ 调研报告）
- 角色的核心使命（如「定义产品方向」→ 产品规划咨询）

映射表（内置已知角色→服务映射）：

### 第三步：六维评估

对每项候选服务，逐项打分：

```python
# 评估模板
{
  "服务名称": "行业调研报告",
  "市场需求": "中高",
  "竞争强度": "黄海",
  "交付难度": 2,
  "单价区间": "500-5000元",
  "用户介入度": "低",
  "月入潜力": "5000-15000元",
  "推荐优先级": "🥇"
}
```

### 第四步：分类入库

按五大类归档，每项服务保留评估数据。分类：

```
📊 数据分析类
🎨 创意设计类
✍️ 内容生产类
🛠️ 技术服务类
📋 咨询培训类
```

### 第五步：排优先级

使用优先级矩阵定位每项服务：

```
高利润 + 低竞争 + 低难度 = 立即做（P0）
高利润 + 中高竞争 + 低难度 = 可以做但需差异化（P1）
低利润 + 红海 = 不做
高利润 + 高难度 + 蓝海 = 条件成熟再做（P2）
```

## 起步建议（P0优先）

根据「高利润 + 低竞争 + 低交付难度」准则：

| 排名 | 服务 | 为什么先做 |
|------|------|-----------|
| P0 | 行业调研报告 | 零成本，我全包，单价高，已经做过验证 |
| P0 | 小红书/B站内容代写 | 需求极大，复购率高，我全包 |
| P0 | 展示页面设计 | HTML无依赖，直接交付，4种风格即用 |
| P0 | Python脚本定制 | 需求最广，入门门槛最低，可引流 |
| P0 | AI工具使用培训 | 市场火热，可线上交付 |

## 重要提醒

- **用户介入度等级要诚实标记**：如果我说「全包」但实际上依赖某个未就绪的环境，必须在菜单中标注
- **逐项验证后再上架**：不要列一个菜单就全部上架，先跑通一个再推下一个
- **价格参考基于 2026年6月 闲鱼/小红书/猪八戒 市场行情**
