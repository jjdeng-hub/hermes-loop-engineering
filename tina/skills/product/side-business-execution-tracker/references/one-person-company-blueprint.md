# 一人公司蓝图 · 实操参考

> 这是 `side-business-execution-tracker` 技能的参考文件。
> 适用于用户想搭建「人 + AI 数字员工」团队结构而非 solo 创业的场景。
> 核心框架来自 Tom & Jerry 的对话实录（Jerry = C++/Python 工程师，Tom = AI 助手）。

---

## 1. 组织架构模板

```
+-------------------------------------------+
|         CEO/决策者（用户本人）               |
|   - 方向判断                                |
|   - 最后拍板                                |
|   - 对外商务（客户沟通、收款、关系维护）      |
+--------------------+----------------------+
                     |
                     v
+-------------------------------------------+
|      COO + CTO（AI 助手）                   |
|   - 战略拆解 -> 执行计划                     |
|   - 内容生产（文案、脚本、翻译、排版）       |
|   - 技术开发（Python/C++/Web/API）          |
|   - 信息调研（竞品、市场、技术趋势）         |
|   - RAG 知识库维护                          |
|   - 自动化运营（cronjob 定时任务、数据汇总）  |
|   - 并发干活（delegate_task 调子代理）       |
+-------------------------------------------+
```

**关键洞察**：用户的时间是稀缺资源，AI 的时间是免费的。设计的核心是**把用户从执行中解放出来，只做判断**。

## 2. 业务模块设计模板

### 模块 A：知识产品（长期核心资产）
- **定位**：把行业/技术知识变成可查询、可售卖的结构化资产
- **产品形态**：个人版（月费 19.9-49.9） + 企业版（年费 5000-20000）
- **AI 完成**：搭建 pipeline、数据清洗入库、自动从 RSS/论坛抓新数据、写查询接口
- **用户完成**：提供原始行业经验、审核输出质量、做客户演示
- **收入曲线**：前3月 0 → 3-6月 500-2000/月 → 6月+ 3000-10000/月

### 模块 B：垂直内容（品牌杠杆）
- **定位**：基于行业经验的内容矩阵，为知识产品和咨询服务引流
- **平台顺序**：小红书（图文）→ B站（视频）→ 知乎（长文）→ 公众号（深度）
- **内容原则**：垂直 > 泛泛（你的独特行业经验是壁垒），不要做 SEO 式无感情内容
- **AI 完成**：从知识库提取知识点/检索网络素材 → 写初稿 → 适配平台风格 → 定时发布 → 数据分析
- **用户完成**：定方向（5分钟）+ 审核技术准确性（3分钟/篇）
- **变现路径**：粉丝 → 私域（微信群）→ 售卖知识库/服务

### 模块 C：轻量服务（快速现金流）
- **定位**：零门槛起步的轻量技术外包，验证市场需求
- **目标客户**：本地小微企业（自动化、数据录入、RPA、客服机器人、爬虫）
- **获客方式**：闲鱼/小红书发布服务帖 + 本地行业群 + 熟人介绍
- **AI 完成**：写脚本、调试、交付文档、回答客户技术问题
- **用户完成**：接单、沟通需求、确认交付、收款
- **收入曲线**：第1月 1000-3000（2-3单） → 2-3月 3000-6000 → 稳定
- **注意**：单子控制在 2-8h/单，超 10h 的除非单价 5000+ 否则不接

### 模块 D：引流工具（免费 → 付费）
- **定位**：免费工具作为流量入口，高级版轻量变现
- **AI 完成**：搭建 Web 界面（FastAPI+前端）、部署、写使用文档
- **用户完成**：确认功能方向、提供行业场景验证

## 3. 数字员工能力映射

| 工具/能力 | 类比传统岗位 | 替换成本/月 |
|-----------|------------|------------|
| AI 助手（主代理） | COO + CTO + 运营 + 内容编辑 | 3-5万 |
| cronjob（定时任务） | 运营专员（每日执行重复任务） | 5000-8000 |
| delegate_task（子代理） | 兼职实习生（按需并行干活） | 时薪30-50 |
| RAG 知识库（数据资产） | 行业研究员 + 资料库管理员 | 1-2万 |

**对比**：传统创业团队3-5人月薪成本约3-5万/月 → 一人公司+AI 的隐形成本≈0（对于有 AI 工具的用户来说）。

## 4. 运营流水线

以"一篇内容"为例的完整流程：

| 步骤 | 谁做 | 耗时 | 产出 |
|------|------|-----|------|
| 周会定方向 | 用户 | 5min | 一句话选题 |
| 提取素材 + 写初稿 | AI | 5-10min | 初稿 + 标题 + 封面文案 |
| 审核技术准确性 | 用户 | 3min | 标注 1-2 处修正 |
| 修复 + 适配平台 + 发布 | AI | 3min | 已发布的帖子 |
| 数据跟踪（1天后） | AI（cronjob） | 0（自动） | 阅读/赞/藏/评 数据 |
| 周报汇总 | AI | 0（自动） | 本周数据 + 趋势分析 |

**用户总投入 / 篇**：8-10 分钟。

## 6. 杠杆选择逻辑

当用户讨论"做什么赚钱"时，按此顺序诊断：

```
1. 用户有什么独特优势？（行业经验 / 技术 / 人脉 / 资源）
2. 这个优势属于哪种杠杆？（知识 / 代码 / 劳动力 / 资本）
3. 用户当前最缺什么？（现金流 / 品牌 / 验证 / 长期壁垒）
4. 推荐哪些快速杠杆先做？（模块C：轻量服务）
5. 推荐哪些长期杠杆养着？（模块A：知识产品 + 模块B：内容）
```

**经验法则**：
- 纯代码杠杆（SEO工具、爬虫、比价器）：窗口期 3-6 个月，AI 生成能力在压缩窗口
- 纯知识杠杆（咨询、培训、课程）：护城河最长，但起步变现慢
- 知识×代码（RAG 知识库工具、行业垂直 SaaS）：最长护城河，推荐优先

## 7. 实操坑位

| 坑 | 表现 | 解决 |
|----|------|------|
| 为工具而工具 | 写了一个没人用的工具 | 先验证需求再写代码：有人付费再开发 |
| 贪多不聚焦 | 同时开3个方向，哪个都没跑通 | 先选1个模块跑通闭环，再扩展 |
| 只想不做 | 反复讨论方向，从不下场 | 设定截止时间：周五前必须发布第一篇/第一单 |
| 高估自己的时间 | 觉得每天能投入4h做副业 | 诚实记录一周时间消耗，实际可用时间往往砍半 |
| 低估AI能做多少 | 让用户写稿/查资料，AI只做辅助 | 反转比例：AI做80%+执行，用户只做判断和审核 |
